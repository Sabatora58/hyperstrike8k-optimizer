# シーズン29対応モデルの作成手順

現行レジェンド＋訓練所ダミーに対応したカスタムYOLOモデルを、
あなたのプレイ画像からファインチューニングで作成します。
クラスは既存モデル互換の 0=Teammate / 1=Enemy（訓練所ダミーはEnemy扱い）。

## 手順

1. **データ収集** — アプリの計測パネルで「学習用データセット収集: ON」にして計測を実行。
   プレイ中2秒ごとに生スクリーンショットが dataset\raw に保存されます（最大1000枚）。
   - 訓練所でダミー相手に5分 → ダミーの学習データ
   - 実戦/Mixtapeを数マッチ → S29レジェンドの学習データ
   - 目安: 合計300枚以上（多いほど良い）
2. **学習** — `train_s29.bat` をダブルクリック。以下を全自動で行います:
   擬似ラベル生成(prep_dataset.py) → PyTorch導入(NVIDIA GPUならCUDA版) →
   apex_v8n.ptを事前学習重みに50エポック学習 → ONNX変換 →
   models\apex.onnx として導入（旧モデルはapex_backup.onnxに退避） →
   DirectML推論ランタイム復旧
3. **確認** — アプリを再起動し、訓練所で計測 → logs\detections の
   プレビュー画像でダミーがEnemy(赤枠)として検出されることを確認。

## 品質を上げたい場合

擬似ラベル（現行モデルによる自動ラベル）には誤りが含まれます。
dataset\apex_s29\train\labels の .txt を labelImg 等で目視修正してから
train_s29.bat を再実行すると精度が上がります。特に:
- ダミーがTeammate(0)と誤ラベルされていたら 1 に修正
- 検出漏れのダミー/敵に枠を追加

## 注意
- 学習はNVIDIA GPU推奨（50エポックで数十分程度）。CPUでも動きますが数時間かかります。
- GPUメモリ不足で落ちる場合は train_s29.bat 内の batch=-1 を batch=8 に変更。
